Temas



Tema 1

¿Qué es el CSS o Hojas de Estilo Cascada?


CSS es un lenguaje que facilita instrucciones para dar estilo a las etiquetas HTML como colores, tipos de letra, tamaños, entre otros, teniendo en cuenta que CSS determina como se van a mostrar los contenidos en la pantalla. CSS viene de sus siglas en inglés "Hojas de estilo cascada" (Cascading Style Sheets).

Avila, M. L. (2020). OVI Aplicando Hojas de Estilo en Cascada CSS. [Objeto_virtual_de_Informacion_OVI]. Repositorio Institucional UNAD. https://repository.unad.edu.co/handle/10596/37679.


Tema 2

Framerworks CSS


Los frameworks son un conjunto estandarizado de conceptos, prácticas y criterios que se usan para abordar un tipo común de problema, que a su vez pueden servir de referencia para ayudar a resolver nuevos problemas similares. También se definen como un paquete compuesto por una estructura de archivos y carpetas de código estandarizado (HTML, CSS, Java Script) que pueden ser utilizados como plataforma para comenzar a construir un sitio web. En este caso los Frameworks CSS son hojas de estilos ya testadas por equipos y diseñadores, listas para ser usadas como base en una maquetación Web, es decir, cómo se veran en la pantalla cada elemento de un sitio web, sin necesidad de escribir todo desde cero. Estas librerías parten de una base sólida, por lo que el diseñador puede centrar más sus esfuerzos en detalles que en la estructura lógica del sitio que en la parte estética del mismo.

Ahora bien la elección del framework para desarrollar una aplicación o sitio web depende de cuan cómodo y entendible sea para el desarrollador,de cuan útil sea para el proyecto y de qué tiempo puede ahorrarse en lograr el resultado final. Actualmente no hay muchas diferencias significativas entre los frameworks, pues son todos bastante completos y no muy complejos de usar. Es por ello que para su seleccionar alguno se podría tener en cuenta los siguientes aspectos: la velocidad de instalación, la complejidad al configurarlos, el consumo de recursos, la integración con otros frameworks y sistemas, y el mejor soporte a largo plazo. Otros elementos a tener en cuenta pudieran ser: persistencia, navegación contextual, cache, Ajax, MVC y la versión.



Riccardi Sabatier, Y., Vega Almeida, R. L., & Miyares Díaz, E. (2018). Aplicación del Responsive Web Design en la creación e implementación del sitio Web del Centro de Histoterapia Placentaria. Revista Cubana de Informática Médica, 10(1), 16-27.
